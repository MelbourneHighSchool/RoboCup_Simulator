Visuals¶

class ev3sim.visual.objects.IVisualElement(**kwargs)¶

A visual element defines some object which can be drawn to the screen, but also can generate a physics object if necessary.

applyToScreen(screen)¶

A method that all visual elements must implement. When invoked, should apply themselves to the screen.

calculatePoints()¶

Called whenever the position or rotation of the object is changed, allowing for any calculation needed to be made.

generateBodyAndShape(physObj, body=None, rel_pos=None)¶

Generates the physics object for this particular visual element. See other implementations of this method for examples.

Parameters

  • physObj (ev3sim.objects.base.PhysicsObject) – The physics object requesting this body and shape.

  • body (pymunk.Body) – If you don’t want a new body generated, supply one.

  • rel_pos (tuple) – If an existing body was specified, then this is the relative position of the shape on the body.

Returns (pymunk.Body, pymunk.Shape)

The generated objects.

initFromKwargs(**kwargs)¶

Initialise the visual object given some extra named arguments (normally provided in the .yaml files).

property position¶

The global position of the visual object - Do not set this when specifying local position of an object! Set the parent object position!

\(0,0\) is the default as centre of the screen, x increasing to the right and y increasing upwards. By convention, please let this position be the centre of your object.

property rotation¶

Rotation of a visual element is float, which should in theory range from \(0\) to \(2\pi\), but should still work outside of those bounds.

sensorVisible: bool¶

Specifies whether the visual object should affect the colour sensor readings.

zPos: float¶

The z Position of the element (higher values for z will be drawn on top of other elements with lower z values)
